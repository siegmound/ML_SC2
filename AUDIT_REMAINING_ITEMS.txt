AUDIT REMAINING ITEMS

Current repository status
- The repository structure is coherent and includes scripts, artifacts, figures, manifests, paper sources, and the final PDF.
- The large datasets are distributed in datasets/ as ZIP archives.
- The training and evaluation scripts expect the extracted CSV files to be present in datasets/.

Dataset policy
- ZIP archives stored in the repository:
  - datasets/starcraft_full_dataset_v3_1_fixed.zip
  - datasets/starcraft_full_dataset_v3_2_combatfix.zip
- These archives must be extracted locally before running any retraining script.
- Expected extracted files:
  - datasets/starcraft_full_dataset_v3_1_fixed.csv
  - datasets/starcraft_full_dataset_v3_2_combatfix.csv
- The extracted CSV files are local runtime inputs and should not be committed back into the repository.

Remaining operational requirement
- A user who wants to rerun training from scratch must extract the dataset ZIP files locally into datasets/ before execution.

Recommended repository cleanup
- Remove tracked __pycache__ content from scripts/ if it is still present in Git.
- Remove tracked extracted CSV files from Git if they were accidentally added after local extraction.
- Keep README.txt, .gitattributes, and .gitignore aligned with the ZIP-based distribution strategy.

Reproducibility note
- The repository is suitable for audit, inspection, and reproduction of preserved artifact-based analyses.
- Full end-to-end retraining becomes available once the dataset ZIP files are extracted into the expected CSV paths inside datasets/.
