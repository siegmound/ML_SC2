TEST 2 METHODOLOGY: CALIBRATION ANALYSIS (XGBOOST VS RANDOM FOREST)

Goal
----
The goal is to evaluate whether XGBoost and Random Forest produce well-calibrated probabilities on the frozen v3_1_fixed test split.

Why calibration matters
-----------------------
Accuracy and ROC-AUC measure discrimination, but they do not tell us whether predicted probabilities are trustworthy.
Since Test 1 already showed that XGBoost has a statistically supported advantage over Random Forest in log loss, calibration analysis checks whether this also appears in probability quality.

Inputs
------
The test uses the prediction files already exported in Test 1:
- xgb_clean_v3_1_fixed_test_predictions.csv
- rf_test_clean_v3_1_test_predictions.csv

These files are merged on:
- replay_id
- time_sec
- y_true

Metrics
-------
For each model, the script computes:
- Brier score
- Log loss
- ROC-AUC
- ECE (Expected Calibration Error)
- MCE (Maximum Calibration Error)

Binning protocol
----------------
Predicted probabilities are divided into 10 bins over [0, 1].
For each bin, we compute:
- number of samples
- mean predicted probability
- observed positive frequency

Outputs
-------
The test produces:
- test2_xgb_calibration_bins.csv
- test2_rf_calibration_bins.csv
- test2_calibration_summary_xgb_vs_rf.json
- test2_reliability_xgb_vs_rf.png
- test2_probability_histograms_xgb_vs_rf.png

Interpretation
--------------
- Lower Brier score is better
- Lower log loss is better
- Lower ECE is better
- Lower MCE is better
- In the combined summary:
  - delta_brier_rf_minus_xgb > 0 favors XGBoost
  - delta_logloss_rf_minus_xgb > 0 favors XGBoost
  - delta_ece_rf_minus_xgb > 0 favors XGBoost

Recommended paper use
---------------------
Use the reliability diagram and Brier/ECE values to support a statement about probability quality.
If XGBoost remains better on calibration-related metrics, this reinforces the Test 1 bootstrap conclusion that its main advantage over Random Forest lies in probabilistic prediction quality rather than raw accuracy.

Package note
------------
This revised freeze distinguishes between two Random Forest artifact lines:
- rf_clean_v3_1_summary.json: aligned benchmark summary retained for the cross-model comparison table;
- rf_test_clean_v3_1_test_predictions.csv / rf_test_clean_v3_1_summary.json: export-preserved RF run used for bootstrap, calibration, and duration-comparison scripts.
The XGBoost prediction export is normalized in this package as xgb_clean_v3_1_fixed_test_predictions.csv for naming consistency.
