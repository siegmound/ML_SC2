TEST 1 METHODOLOGY: PAIRED REPLAY-BOOTSTRAP SIGNIFICANCE TEST (XGBOOST VS RANDOM FOREST)

Goal
----
The goal is to assess whether the small observed performance difference between XGBoost and Random Forest on the v3_1_fixed dataset is statistically robust.

Why replay-level bootstrap?
---------------------------
The test set contains many snapshots from the same replay. These snapshots are not independent.
If we resampled individual rows directly, we would artificially overstate confidence.
Therefore, the bootstrap unit is the replay_id, not the individual snapshot.

Protocol
--------
1. Train XGBoost on v3_1_fixed using the same group-aware train/validation/test protocol already used in the benchmark.
2. Save the test predictions in:
   - xgb_clean_v3_1_fixed_test_predictions.csv
3. Train Random Forest on v3_1_fixed using the same group-aware train/validation/test protocol.
4. Save the test predictions in:
   - rf_test_clean_v3_1_test_predictions.csv
5. Merge the two test prediction files by:
   - replay_id
   - time_sec
   - y_true
6. Run a paired bootstrap over replay_id:
   - sample test replays with replacement
   - include all snapshots for each sampled replay
   - recompute metrics for each bootstrap sample

Metrics
-------
For each bootstrap replicate we compute:
- accuracy
- ROC-AUC
- log loss

Differences are defined as:
- diff_accuracy_xgb_minus_rf
- diff_auc_xgb_minus_rf
- diff_logloss_rf_minus_xgb

Interpretation
--------------
- Positive accuracy difference favors XGBoost
- Positive AUC difference favors XGBoost
- Positive logloss difference favors XGBoost because it is RF logloss minus XGBoost logloss

Outputs
-------
The test produces:
- test1_xgb_vs_rf_bootstrap_samples.csv
- test1_xgb_vs_rf_bootstrap_summary.json

Recommended interpretation
--------------------------
Use the 95% bootstrap confidence intervals:
- if the CI excludes 0, the difference is statistically supported
- if the CI includes 0, the difference is too small to claim a robust advantage

Package note
------------
This revised freeze distinguishes between two Random Forest artifact lines:
- rf_clean_v3_1_summary.json: aligned benchmark summary retained for the cross-model comparison table;
- rf_test_clean_v3_1_test_predictions.csv / rf_test_clean_v3_1_summary.json: export-preserved RF run used for bootstrap, calibration, and duration-comparison scripts.
The XGBoost prediction export is normalized in this package as xgb_clean_v3_1_fixed_test_predictions.csv for naming consistency.
