TEST 4 METHODOLOGY: MLP CALIBRATION AND 3-WAY CALIBRATION COMPARISON

Goal
----
The goal is to complete the probabilistic comparison by evaluating MLP calibration on the same frozen v3_1_fixed test split, and then compare all three final models:
- XGBoost
- Random Forest
- MLP PyTorch

Protocol
--------
1. Re-run the final MLP configuration on v3_1_fixed using the same group-aware train/validation/test split already adopted in the benchmark.
2. Export the MLP test predictions to:
   - mlp_torch_gpu_v3_1_export_test_predictions.csv
3. Merge the three prediction files:
   - xgb_clean_v3_1_fixed_test_predictions.csv
   - rf_test_clean_v3_1_test_predictions.csv
   - mlp_torch_gpu_v3_1_export_test_predictions.csv
4. Compute calibration tables over 10 bins for each model.
5. Compute:
   - Brier score
   - Log loss
   - ROC-AUC
   - ECE
   - MCE
6. Generate a 3-way reliability diagram.

Outputs
-------
- test4_xgb_calibration_bins.csv
- test4_rf_calibration_bins.csv
- test4_mlp_calibration_bins.csv
- test4_reliability_xgb_rf_mlp.png
- test4_calibration_summary_xgb_rf_mlp.json

Interpretation
--------------
This test is optional from the paper-completion perspective, but useful to show whether the probabilistic quality ranking remains:
XGBoost > Random Forest > MLP, or whether MLP is closer to RF than expected in calibration terms.

Package note
------------
This revised freeze distinguishes between two Random Forest artifact lines:
- rf_clean_v3_1_summary.json: aligned benchmark summary retained for the cross-model comparison table;
- rf_test_clean_v3_1_test_predictions.csv / rf_test_clean_v3_1_summary.json: export-preserved RF run used for bootstrap, calibration, and duration-comparison scripts.
The XGBoost prediction export is normalized in this package as xgb_clean_v3_1_fixed_test_predictions.csv for naming consistency.
