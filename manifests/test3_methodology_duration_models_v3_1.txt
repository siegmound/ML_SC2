TEST 3 METHODOLOGY: COMPARATIVE ACCURACY VS MATCH DURATION

Goal
----
The goal is to compare how the three final models behave across match-duration bins on the frozen v3_1_fixed test split:
- XGBoost
- Random Forest
- MLP PyTorch

Why this test matters
---------------------
Previous experiments showed that all models perform better in the standard early/mid-game regime and degrade on longer matches.
This test consolidates that observation in a single final figure/table.

Protocol
--------
1. For XGBoost and Random Forest, use the saved test prediction CSVs:
   - xgb_clean_v3_1_fixed_test_predictions.csv
   - rf_test_clean_v3_1_test_predictions.csv
2. For each replay in the test set, estimate match duration as:
   - max(time_sec) among that replay's test snapshots
3. Assign each snapshot to the duration bin of its replay:
   - duration_min_bin = floor(match_length_sec / 300) * 5
4. Compute, for each bin:
   - accuracy
   - n_samples
   - n_replays
5. For MLP PyTorch, load the already saved corrected duration table:
   - mlp_torch_gpu_v3_1_accuracy_vs_duration.csv
6. Merge the three model tables into one final comparison table.

Outputs
-------
The test produces:
- test3_duration_comparison_v3_1.csv
- test3_duration_comparison_v3_1.png
- test3_duration_comparison_v3_1_sample_counts.png
- test3_duration_comparison_v3_1_summary.json

Interpretation
--------------
- If all three models show a similar drop after 15–20 minutes, the bottleneck is likely not the classifier family but the representation of late-game states.
- If XGBoost and RF remain close across bins while both outperform MLP, this reinforces the conclusion that tree models are the most effective choice for this engineered tabular representation.

Package note
------------
This revised freeze distinguishes between two Random Forest artifact lines:
- rf_clean_v3_1_summary.json: aligned benchmark summary retained for the cross-model comparison table;
- rf_test_clean_v3_1_test_predictions.csv / rf_test_clean_v3_1_summary.json: export-preserved RF run used for bootstrap, calibration, and duration-comparison scripts.
The XGBoost prediction export is normalized in this package as xgb_clean_v3_1_fixed_test_predictions.csv for naming consistency.
